PENDAHULUAN
-----------
Setiap Line (1-4) memiliki dua file input data:
  - INPUTC.jsx  => Form input untuk proses BLOWING-FILLING-STERIL (Zone C)
  - INPUTF.jsx  => Form input untuk proses FINISHING/PACKAGING (Zone F)

Keduanya menggunakan arsitektur yang sama (spreadsheet berbasis custom React),
namun berbeda secara konteks data, struktur kolom, dan logika kalkulasi
sesuai dengan kebutuhan proses masing-masing zona.

==============================================================================
TEMUAN & REKOMENDASI
==============================================================================

1. [INKONSISTENSI] Line 1 INPUTF memiliki infrastruktur virtual scrolling dan
   requestAnimationFrame yang tidak dimiliki Line 2, 3, 4. Sebaiknya
   distandarisasi jika semua Line bisa memiliki data besar.

2. [INKONSISTENSI] parseToYMD di Line 1 lebih robust (handle multiple format
   tanggal) dibanding Line 2/3/4 yang hanya handle DD/MM/YYYY. Risiko:
   data dari sumber lain bisa gagal di-parse di Line 2/3/4.

3. [LINE 3 UNIK] Line 3 memiliki perbedaan terbesar:
   - Hanya 1 produk (25 ML)
   - Kolom lebih sedikit (40 vs 52 untuk INPUTC, 46 vs 52 untuk INPUTF)
   - Shift & Group menggunakan dropdown select
   - "Reject Before Steril" bisa diinput manual (Line 1/2/4 readOnly)
   - Default Utuh? dan Planned tidak di-preset
   - Proses Downtime INPUTC tidak ada Blowing (mesin berbeda)
